Detailed Description

Non-Owning string view.

bench::String is non-owning, UTF-8, non-zero-terminated string view. It has two register-sized members - data and length.

It's usually passed to functions by value. If a function accepts a String by value, the caller must guarentee, that the String will live until the function returns (this basically means that you won't delete it from another thread).

String does not provide a storage mechanism. If you wish store a string for long-term usage you must copy it onto memory you own:

  • You can use HeapString, a class similar to the STL std::string.
  • You can copy it onto an Arena via Arena::InternString - the resulting string will live until the Arena is reset or destroyed.

For UTF-16 string support see Arena::InternString and Arena::InternWideCString. The idiomatic pattern when working with Windows APIs is to do all string processing with bench::String functionality, and convert to a wide-string on a scratch Arena right before
