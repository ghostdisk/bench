◆ FileCreateDisposition

enum class bench::FileCreateDisposition
strong

Specifies how File::Open behaves when opening or creating a file.

These values match the Win32 API creation disposition values. See Microsoft docs: https://learn.microsoft.com/en-us/windows/win32/api/fileapi/nf-fileapi-createfilew

Enumerator
CREATE_NEW 

If the file exists: Fail.


If it does not: Create it and open it.

CREATE_ALWAYS 

If the file exists: Open it and truncate it to zero length.


If it does not: Create it and open it.

OPEN_EXISTING 

If the file exists: Open it.


If it does not: Fail.

OPEN_ALWAYS 

If the file exists: Open it.


If it does not: Create it and open it.

◆ VirtualMemoryProtection

enum class bench::VirtualMemoryProtection : U8
strong

Memory protection access bitmask.

If no other flags are set, memory range will be unaccessible. Memory range will have readable. Memory range will have writable. On x86 you can't have write-only memory, you need READ as well! Memory range will be executable.
